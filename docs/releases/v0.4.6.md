# Kyrei v0.4.6 — First chat just works

Patch release: after install, **connect a provider → pick a model → chat**.  
No more sticky empty session on the unready seed.  
Also ships the v0.4.5 train that was not published on GitHub yet (Wave H memory + Arch packaging).

---

## Highlights

| | |
| --- | --- |
| **First-run chat** | Adding a ready provider auto-activates when the seed default has no credentials, rebinds idle chats, and falls back at prompt time if the session still pointed at the stub. |
| **Settings UX** | “Use as default” turns **on** when no ready provider exists; major templates ship seed model IDs so you can Save without Discover. |
| **Arch install** | Pacman package no longer depends on dead `libappindicator`; Wayland gets `ELECTRON_OZONE_PLATFORM_HINT=auto` + display guards. |
| **Memory (0.4.5 train)** | LTM pin / SUPERSEDE, `memory_ask`, MMR recall, curator capture signals — already in the tree, now in this release channel. |

---

## What’s fixed (this patch)

| Area | Change |
| --- | --- |
| **Sticky session** | Idle sessions snapshotted to `default-openai-compatible` rebind when a ready default appears (`rebindUnreadyIdleSessions`). |
| **First ready wins** | Create / update / secret paths auto-activate the first ready provider while the active default is still unready. |
| **Prompt fallback** | Unready or missing session target falls back to the ready active default (`provider_credentials_required` / missing id only — not account eligibility). |
| **UI default** | `shouldDefaultUseAsDefault` pre-checks “Use as default” on first configure. |
| **Template seeds** | OpenAI, Anthropic, Gemini, OpenRouter, DeepSeek, Ollama, LM Studio include default model ids (templates v4). |
| **Arch + Wayland** | Dropped `libappindicator`; launcher rejects root / `--no-sandbox` / missing display; Ozone auto on Wayland. |

## First launch (expected)

1. Install the **v0.4.6** artifact for your OS (see below).
2. Open Kyrei → **Settings → Providers**.
3. Pick a catalog provider (e.g. OpenAI) → paste API key → **Save**  
   (default checkbox is on when nothing is ready yet).
4. Close settings and **type in the existing chat** — no need to invent a new session.
5. Choose a **workspace** when you want file tools / project memory.

### What still needs the user (not bugs)

| Item | Why |
| --- | --- |
| API key (or local Ollama/LM Studio) | Chat needs a ready model endpoint |
| Linux keyring | Secrets fail closed without Secret Service — install `gnome-keyring` / KWallet, re-login |
| Workspace folder | Coding tools, LTM, plan files need a project root |

## Install out of the box

Verify every artifact against **`SHA256SUMS.txt`** in this release before first run.

### Windows (x64)

| Artifact | Use |
| --- | --- |
| `Kyrei-Setup-0.4.6.exe` | Installer (desktop + Start Menu) |
| `Kyrei-Portable-0.4.6.exe` | Portable — no install |

> Builds are not commercial code-signed yet. SmartScreen may warn once.

### macOS

| Artifact | Arch |
| --- | --- |
| `Kyrei-0.4.6-x64.dmg` / `.zip` | Intel |
| `Kyrei-0.4.6-arm64.dmg` / `.zip` | Apple Silicon |

> Not notarized yet. Gatekeeper: right-click → Open, or **Privacy & Security**.

### Linux

| Artifact | Notes |
| --- | --- |
| `Kyrei-0.4.6-*.AppImage` | x64 (and arm64 when published) |
| `Kyrei-0.4.6-*.deb` | Debian/Ubuntu (`libsecret` + keyring recommends) |
| `Kyrei-0.4.6-*.pkg.tar.zst` | **Arch — use this build or newer** (no `libappindicator`) |

```bash
# Arch
sudo pacman -U Kyrei-0.4.6-*.pkg.tar.zst
# optional but recommended for API keys
sudo pacman -S gnome-keyring   # or kwallet on Plasma
```

Prefer DEB/pacman over AppImage when possible. **Do not** run with `sudo kyrei` or `--no-sandbox`.  
See [`docs/linux-launch.md`](../linux-launch.md).

## Upgrade notes

- From **0.4.4 / 0.4.5**: safe in-place upgrade.
- Existing provider secrets keep working with the same OS key store.
- If an open chat still looks “stuck”, send a message once (runtime rebind) or open a new session after upgrade — idle rebind also runs when you save a ready provider.
- **Arch users on pre-0.4.5 packages:** replace the install entirely; old deps referenced a removed Arch package.
