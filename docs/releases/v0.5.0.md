# Kyrei v0.5.0 — Memory that stays coherent, teams that can disagree safely

Kyrei's project memory is now deterministic, self-cleaning, and safer to share across agents. Team mode also gains structured comparison, evidence-aware conflicts, and durable human clarification instead of silently guessing when agents disagree.

---

## Highlights

| | |
| --- | --- |
| **Clean long-term memory** | Reindexing refreshes the project graph, prunes stale Tier-A and vector records, canonicalizes workspace identity, and keeps chunked embeddings searchable. |
| **Deterministic project knowledge** | Project files are ordered consistently, generated `output/` content is ignored, and explicit rebuilds produce a stable index without stale documents. |
| **Team comparison** | Parallel roles return structured claims with confidence, evidence, and provenance; Kyrei classifies the result as `converged`, `needs_verification`, or `needs_human`. |
| **Human-in-the-loop pipelines** | Blocking questions survive persistence, pause at the approval boundary, and carry the operator's answer into downstream stages as a provenance-tagged artifact. |
| **Safer settings updates** | Workspace configuration is saved transactionally so memory status and concurrent settings edits do not overwrite each other. |

---

## Memory improvements

- Prune stale project, vault, session, and vector records during reindex.
- Split large documents into bounded embedding chunks and search across those chunks.
- Refresh `project_index` during explicit rebuilds and keep project traversal deterministic.
- Reindex when vault, planning, or long-term-memory projections change — not on every ordinary chat turn.
- Normalize workspace identity across Windows path casing and separators.
- Keep the durable file backend available when the development runtime has a native SQLite ABI mismatch.
- Show indexing readiness and failures more accurately in Settings.

---

## Team and pipeline improvements

- Team members can expose clarification requests in their structured output.
- Consensus and department runs compare related role results without treating unrelated tasks as contradictions.
- Comparisons retain claims, conflicts, confidence, evidence, and provider/model provenance.
- Low lexical similarity is treated conservatively as a verification signal, not proof of a semantic conflict.
- The acting orchestrator consolidates blocking questions before asking the human.
- Pipeline artifacts persist clarification requests with validation and defensive copying.
- Human answers are forwarded downstream as explicit `human/operator` decisions.

---

## Upgrade notes

- Safe in-place upgrade from **v0.4.7**; existing sessions, provider secrets, and project memory remain compatible.
- Open each active workspace once so Kyrei can refresh canonical workspace tags and remove stale index records.
- If the local development checkout falls back from SQLite, rebuild `better-sqlite3` for the runtime you are launching. Packaged desktop artifacts are rebuilt for Electron by the release workflow.

## Install

Download the artifact for your platform and verify it against **`SHA256SUMS.txt`**.

| Platform | Artifacts |
| --- | --- |
| Windows x64 | Setup and portable `.exe` |
| macOS Intel / Apple Silicon | `.dmg` and `.zip` |
| Linux x64 | AppImage, `.deb`, and Arch `.pkg.tar.zst` |
| Linux arm64 | AppImage and `.deb` |

Prefer native deb/pacman packages over AppImage where available. Never run Kyrei as root.

## Known development constraint

Node and Electron use different native module ABIs. A single development copy of `better-sqlite3` cannot simultaneously target Node/Vitest and Electron; rebuild it for the runtime being launched. This does not affect the packaged artifacts produced by the release workflow.
