# Kyrei v0.7.1 — Agents that wait instead of dying

Kyrei 0.7.1 removes artificial time-based task failures from read-only agents,
Team roles and provider streams. Slow work now remains observable and
cancellable without being mistaken for dead work. This patch also repairs the
native SQLite release pipeline so packaged memory uses the Electron ABI rather
than silently falling back to files.

## Long-running agents and teams

- Idle and maximum-runtime thresholds are advisory leases. Crossing one emits
  `recovering`, keeps the task alive and continues waiting for the provider.
- Fresh tool or model progress moves the task back to `running`. A late but
  valid provider result is accepted instead of being replaced with a timeout.
- Queue time does not consume a Team role's execution lease, and the former
  hidden five-minute clamp is gone. Explicit values up to one hour are retained.
- Parent/user cancellation remains immediate and terminal. Real provider errors
  still surface normally; Kyrei does not blindly replay write-heavy work that
  could duplicate side effects.
- Provider-account capacity remains leased until the physical request settles,
  including transports that are slow to observe cancellation.

## Provider streaming without false timeouts

- Header and body-inactivity limits are separate and configurable. Both default
  to `0`, meaning a slow gateway waits until it responds or the user cancels.
- Raw response chunks and SSE heartbeat traffic renew activity even when no
  visible answer token has arrived yet.
- Streaming now respects backpressure, keeps the per-origin connection slot
  until the body closes, and preserves external `AbortError` cancellation.
- Users who prefer fail-fast networking can set explicit header or inactivity
  limits in Capacity settings; `0` always disables the corresponding timer.

## Accurate lifecycle in the interface

- `recovering`, `partial` and `interrupted` are no longer flattened into
  misleading `failed` or `completed` states between engine, gateway and UI.
- Team settings expose independent idle and maximum-runtime thresholds with
  clear advisory wording in Russian and English.
- Capacity settings expose first-header and stream-inactivity controls, including
  the unlimited default required for slow self-hosted providers.

## Native SQLite release repair

- The release packager force-rebuilds `better-sqlite3` for the installed
  Electron version before packaging, then restores the workspace binary for
  Node development even if packaging fails.
- GitHub Actions uses the same ABI-safe packager on Windows, macOS, Linux and
  Arch instead of bypassing it with a direct `electron-builder` call.
- Windows smoke verification confirmed Node ABI 137 in the workspace, Electron
  ABI 148 in the package, a native in-memory SQLite query, engine import and a
  packaged gateway `/health` response.

## Upgrade notes

Install normally through Kyrei's updater or download the package for your
platform. Sessions, memory, imported knowledge, provider secrets, Team state,
indexes and UI preferences remain in the user-data directory. Existing explicit
network timeouts remain preserved; new installations default to unlimited
provider waiting with explicit cancellation available at all times.
