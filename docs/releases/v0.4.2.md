# Kyrei v0.4.2

Local-first desktop coding agent: two-stage context compression, session forks, memory/skills curators, and a safer Linux launch — without silent rewrites of `SKILL.md` or `MEMORY.md`.

---

## Highlights

### Agent runtime

| Area | What you get |
| --- | --- |
| **Context compression** | Stage A: tool-output CCR prune. Stage B: REFERENCE-ONLY middle summary for the model only (full chat history stays in the UI). Dual-trigger: `max(local estimate, last-step provider usage)`. |
| **Session fork / lineage** | Branch a chat with `parentSessionId` / `rootSessionId`; sidebar nests children; parent is immutable; pending approvals are cleared on the fork. |
| **Soft-archive** | Hide sessions without deleting messages (hybrid FTS retained). Restore, permanent delete, and batch curate live under **Settings → Sessions**. |
| **Memory curator** | On archive by default (`apply_safe`: notes / LTM / handoff). `MEMORY.md` only via apply-all or manual apply. Runs **in the background** with a hard timeout — archive never blocks the UI. |
| **Skills curator** | **Opt-in, default OFF.** Heuristic stale/dup/thin scan; optional LLM patch as a **proposal** only. Safe apply disables stale skills — never silent `SKILL.md` rewrites. |
| **Approval → rule** | Always allow / Always deny → exact path or command rules. Absolute or empty paths never become tool-wide allows. |
| **Personality catalog** | helpful · concise · technical · teacher · reviewer · implementer + custom text. |
| **Images** | `auto` / `native` / `text` routing; paste and drop in the composer. |
| **Also** | Session mirror dual-write, hybrid SQLite FTS, MCP opt-in, cron, supervised hunk file review, team/pipeline, EN/RU UI. |

### Linux / secrets (desktop shell)

- Clear failure when launched as root (`sudo kyrei`) — install as root, run as your user.
- Sandbox stays on; `--no-sandbox` (including `=value`) is blocked.
- When the OS secret store is unavailable on Linux/Arch, Kyrei points to Secret Service / `gnome-keyring` instead of writing keys in plain text.

### Product boundaries (intentional)

Not in this release: remote/cloud Hermes gateway, pets, MoA, Computer Use, silent post-turn skill/memory rewrites, auto-updater, full multi-backend PTY.

## Download & install

Verify every file against **`SHA256SUMS.txt`** in this release before first run.

### Windows (x64)

| Artifact | Use |
| --- | --- |
| `Kyrei-Setup-0.4.2.exe` | Installer (desktop + Start Menu shortcuts) |
| `Kyrei-Portable-0.4.2.exe` | Portable — no install |

1. Run Setup or Portable.  
2. **Settings → Providers** — add an API key (or Kiro connector).  
3. Choose a workspace folder.  
4. New session → send a message.

> Windows builds are not yet commercial code-signed. SmartScreen may warn on first launch.

### macOS

| Artifact | Arch |
| --- | --- |
| `Kyrei-0.4.2-x64.dmg` / `.zip` | Intel |
| `Kyrei-0.4.2-arm64.dmg` / `.zip` | Apple Silicon |

> Not notarized yet. If Gatekeeper blocks: right-click → Open, or allow the app under **System Settings → Privacy & Security**.

### Linux

| Artifact | Notes |
| --- | --- |
| `Kyrei-0.4.2-*.AppImage` | x64 and arm64 |
| `Kyrei-0.4.2-*.deb` | Debian/Ubuntu |
| `Kyrei-0.4.2-*.pkg.tar.zst` | Arch (`pacman -U`) |

Prefer **DEB/pacman** over AppImage when possible (sandbox / user namespaces). Do not run the app with `sudo`.

## Safe defaults

| Feature | Default |
| --- | --- |
| Skills curator | **Off** |
| Messaging webhook | **Off** |
| Postgres memory backend | **Off** (SQLite) |
| Memory curator on archive | **On** (`apply_safe`, background) |
| Silent `SKILL.md` / full `MEMORY.md` rewrite | **Never** |
