# Kyrei v0.4.2 — безопасный запуск Linux и понятные ключи провайдеров

Этот патч делает запуск Kyrei на Linux и Arch предсказуемым, не ослабляя
защиту Electron.

## Главное

- **Понятный запуск без root.** Команда `sudo kyrei` больше не приводит к
  аварийному сообщению Electron: Kyrei завершится до старта движка и объяснит,
  что `sudo` нужен только для установки пакета, а запускать приложение нужно
  обычной командой `kyrei`.
- **Sandbox остаётся включённым.** Kyrei блокирует `--no-sandbox`, в том числе
  варианты с присвоением значения. Если AppImage не может использовать user
  namespaces, он выдаёт безопасную инструкцию перейти на DEB/Arch-пакет либо
  включить namespaces, а не запускается с отключённой песочницей.
- **Понятная настройка API-ключей.** При недоступном защищённом хранилище
  провайдеров Kyrei определяет платформу и на Linux/Arch показывает конкретный
  путь: настроить Secret Service/keyring, например установить
  `gnome-keyring`, войти в графическую сессию заново и повторить сохранение.
  Ключи по-прежнему никогда не сохраняются открытым текстом.
- **Стабильные checkpoint-проверки.** Тесты восстановления теперь ждут
  фактического окончания agent-turn, а не раннего появления черновика
  сообщения.

## Платформы и файлы

- Windows x64: Setup и Portable.
- macOS: DMG и ZIP для Intel и Apple Silicon.
- Linux x64/arm64: AppImage и DEB.
- Arch Linux x64: `.pkg.tar.zst` для `pacman`.

В каждом релизе приложен `SHA256SUMS.txt` для проверки скачанных файлов.

> Пакеты Windows пока не подписаны коммерческим сертификатом, а пакеты macOS
> пока не notarized. Перед первым запуском сверяйте SHA-256 из этого релиза.

## Проверка качества

- Полный gate: TypeScript, JavaScript, i18n и Vitest.
- Тесты Linux launcher для root, `--no-sandbox=*` и упаковочного hook.
- Нативная упаковка и публикация всех платформ выполняются в GitHub Actions.
