# Kyrei v0.7.4 — корпоративный шлюз и управляемый доступ

Kyrei 0.7.4 позволяет владельцу развернуть шлюз на своём рабочем компьютере
или в корпоративной VPN-сети и безопасно предоставить сотрудникам доступ к
настроенным моделям без передачи ключей самих провайдеров.

## Корпоративный шлюз

- В настройках появилась отдельная страница **«Корпоративный шлюз»**.
- Владелец может включить совместимый OpenAI API на loopback или в локальной
  сети, после чего интерфейс показывает точный адрес вида
  `http://<IP-владельца>:<порт>/v1` и позволяет скопировать его.
- Для сотрудника создаётся отдельный ключ `kyrei_at_…`: ему можно назначить
  понятное имя, срок действия и строго определённые маршруты
  `providerId/modelId`.
- Сотрудник видит через `/v1/models` только разрешённые ему модели. Попытка
  обратиться к другой модели отклоняется до отправки запроса провайдеру.
- Владелец может изменить scope и срок действия, отключить, перевыпустить или
  отозвать ключ, а также видеть число запросов, токены и стоимость по каждому
  ключу.

## Подключение сотрудника

1. Владелец добавляет аккаунты и пулы провайдеров в Kyrei, включает LAN-режим
   шлюза и перезапускает Kyrei.
2. В разделе «Корпоративный шлюз» он создаёт ключ сотрудника и передаёт только
   LAN/VPN-адрес шлюза и этот ключ.
3. Сотрудник добавляет в Kyrei Custom OpenAI-compatible provider с полученным
   базовым URL и ключом. Ключи OpenAI, Anthropic, Z.AI и других провайдеров
   остаются только на машине владельца.

Не открывайте LAN-порт напрямую в публичный интернет. Для внешней работы
используйте VPN либо HTTPS reverse proxy с firewall. Не подключайте владелецкий
экземпляр Kyrei к его же `/v1` адресу: для проверки сотруднического ключа
используйте второй профиль, отдельную установку или совместимый клиент.

## Провайдеры и надёжность

- Сохранена поддержка пулов аккаунтов одного провайдера с балансировкой,
  health/cooldown и session affinity.
- В шлюзовый доступ добавлена строгая фильтрация моделей и корректный отказ для
  истёкшего ключа.
- После интеграции `v0.7.2` ChatGPT/Codex connector продолжает использовать
  актуальную конфигурацию проекта и MCP, а API повторного запуска агентов не
  потерян при слиянии изменений.
