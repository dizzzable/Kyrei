# Kyrei v0.7.0 — Knowledge Fabric and resilient teams

Kyrei 0.7 connects project memory, imported agent sessions, Skills, Team runs
and safe improvement proposals into one visible Knowledge Fabric. It also
removes several false-stop and false-provider-ban paths reported during the
0.6 series.

## Memory Atlas

- A new three-pane Memory Atlas adds a source/directory tree, interactive
  graph and contextual inspector. The canvas supports pan, cursor-anchored
  zoom, keyboard navigation, fit/reset controls, pinned nodes and a minimap.
- Project files, durable memory, plans, handoffs, sessions, imported
  documentation, Skills, linked Skill documents and evolution candidates use
  stable identities in one projection.
- Folder imports preserve their relative hierarchy and reject traversal or
  binary/unsupported content.
- Atlas viewport, pane widths, expanded directories and pinned positions are
  persisted per workspace without writing UI noise into the repository.
- Skills can be sent directly from the inspector to the next Composer request.

## Skills and imported agent knowledge

- Every enabled Skill is metadata-searchable; the former top-32 visibility
  ceiling is gone. Kyrei injects only a bounded catalog preview and lazily reads
  a relevant `SKILL.md` when the model requests it.
- Skill compatibility, source, linked documents and content digest are visible
  in the Atlas and available to single-agent and Team runtimes.
- Codex CLI, Claude Code, Hermes, OpenCode, generic agent JSON, Kyrei exports
  and Markdown histories are normalized before they enter Kyrei. Foreign tool
  payloads are treated as untrusted evidence instead of being passed into the
  provider `ModelMessage[]` schema.

## Runtime resilience

- Read-only and Team agents now use a progress-aware idle lease plus a separate
  hard runtime ceiling. Observable tool/step progress renews the lease, queued
  work does not consume it, and legacy 90-second settings migrate to 180
  seconds with a 30-minute hard ceiling.
- Useful evidence from an exhausted child is returned as `partial`; one child
  failure no longer kills unrelated agents or the parent chat. Durable,
  redacted checkpoints surface legal retry/cancel actions after interruption.
- Subscription Shield timeouts are classified as retryable provider attempts
  and cannot directly become an invented terminal handoff.
- Streaming no longer locks the user to the bottom of the chat. Scrolling up
  pauses follow mode; `Jump to latest` resumes it without stopping the agent.
- Provider-exposed reasoning is stored as ordered, stable segments. Empty
  reasoning blocks and internal AUTO/recovery markers are not shown, copied,
  spoken or exported as the answer.

## Providers, MCP and local services

- Custom providers are adapter-based, not brand-allowlisted. A public HTTPS
  literal IP is supported; exact-origin public HTTP requires an explicit
  opt-in. SSRF, reserved-address, DNS-rebinding, redirect and credential-leak
  protections remain enforced.
- Ambiguous custom-proxy 401 responses require repeated evidence before an
  account enters `auth-required`; soft 403 and transient network failures use
  cooldown only. Runtime account state can be reset without deleting provider
  configuration or secrets.
- MCP settings support both stdio and Streamable HTTP servers with bounded,
  redacted connectivity diagnostics.
- Team Postgres works out of the box through the bundled PGlite socket runtime;
  an external Postgres connection remains available.
- Kyrei Memory is now a built-in offline store. It initializes locally without
  Bun, GitHub, PATH, browser login, embedding service or provider key; external
  GBrain CLI remains an optional compatibility mode.
- Index, session-mirror, GBrain and optional-source health checks share
  single-flight hysteresis and keep the last good snapshot, preventing rapid
  available/unavailable flicker. Incremental session projection no longer
  prunes unrelated sessions.

## Appearance, updates and safe evolution

- Chat backgrounds follow the selected/imported theme by default; the Kyrei
  peony artwork is an explicit persistent option.
- The Composer keeps model/mode/agent controls and microphone/send actions
  aligned on the right while the input grows upward.
- The update indicator glints only when a release is available. Installed
  updates still require explicit confirmation and replace the complete app
  bundle so engine/native dependency versions cannot be mixed.
- Evolution candidates are append-only, redacted and proposal-first. Missing
  evaluation/apply infrastructure blocks promotion only; it never mutates the
  active prompt, Skill, memory policy or user task in-band.

- Engine and renderer TypeScript checks, JS syntax checks and RU/EN hardcode

- Windows package preparation and installed-build smoke are performed from the

## Upgrade notes

The update is in place. Sessions, imported histories, provider secrets,
project memory, indexes, Skill roots and UI preferences remain in the user data
directory. Legacy 90-second child timeout values are migrated automatically;
explicit custom timeout/runtime settings remain bounded and are preserved.
