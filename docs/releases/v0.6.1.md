# Kyrei v0.6.1 — durable memory sync and usable MCP setup

This patch release turns two previously fragile integration paths into
observable, recoverable workflows: the local chat-session mirror and MCP
server configuration.

## Fixed

| | |
| --- | --- |
| **Resumable session mirror** | Full chat mirroring now has a durable cursor, serializes concurrent writes, reports progress in Settings, and resumes safely after an interrupted shutdown instead of restarting or leaving a torn mirror. |
| **Truthful prompt inspection** | Settings can show Kyrei's redacted baseline system prompt with version, tool manifest, and explicit omissions. It does not misrepresent a static preview as a full turn transcript. |
| **Working Streamable HTTP MCP** | The config boundary now preserves a Streamable HTTP `url` instead of rejecting it as a missing stdio `command`. The MCP editor supports local stdio and remote HTTP servers directly; advanced JSON remains available for headers, environment variables, and working directories. |

## Upgrade notes

- Safe in-place upgrade from **v0.6.0**. User sessions, secrets, and memory
  directories remain untouched.
- If a mirror rebuild was interrupted, open **Settings → Memory**; Kyrei will
  resume it automatically and the status shows its durable progress.
- For remote MCP use **Streamable HTTP** and the endpoint URL. Connection
  checks redact sensitive values; credentials remain user-managed.
