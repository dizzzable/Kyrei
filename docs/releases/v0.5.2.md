# Kyrei v0.5.2 — Embedded Team Postgres and Linux secrets out of the box

This patch release makes Team mode and Linux credential storage self-contained
for native desktop packages. It keeps local-first defaults while removing the
manual PostgreSQL and keyring setup that previously blocked a clean first run.

## Highlights

| | |
| --- | --- |
| **Built-in Team database** | Team mode starts a persistent PGlite PostgreSQL-compatible runtime on loopback, scoped per workspace, with pgvector enabled when available. No system PostgreSQL service or Docker setup is required. |
| **External Postgres remains supported** | A user-supplied connection string is preserved and never replaced by the built-in runtime; external Postgres remains the option for sharing memory across machines. |
| **Linux secrets installed by native packages** | DEB and Arch packages now hard-depend on `libsecret` and `gnome-keyring`. Kyrei forces the generic Secret Service backend and refuses Electron's unprotected `basic_text` fallback. |
| **Actionable Linux diagnostics** | Provider setup explains Wayland session unlock, shows the encrypted envelope path, and points to Seahorse/KWallet entries. AppImage limitations are documented clearly. |
| **Node 24 alignment** | Development and release documentation now matches the enforced Node.js `>=24` engine. |

## Upgrade notes

- Safe in-place upgrade from **v0.5.1**; provider secrets, sessions, and
  workspace memory files remain on disk.
- Switching to Team mode automatically provisions the built-in database when
  no external Postgres URL is configured.
- The built-in database is loopback-only and persistent per workspace. It is
  not a multi-machine PostgreSQL server.
- On Linux, prefer the native DEB or Arch package for automatic keyring
  installation. Pure Wayland compositors still need a graphical login/PAM
  session that starts and unlocks Secret Service.
- AppImage continues to use the host's Secret Service and cannot install a
  system DBus/PAM keyring integration by itself.

**`SHA256SUMS.txt`**.

| Platform | Artifacts |
| --- | --- |
| Windows x64 | Setup and portable `.exe` |
| macOS Intel / Apple Silicon | `.dmg` and `.zip` |
| Linux x64 | AppImage, `.deb`, and Arch `.pkg.tar.zst` |
| Linux arm64 | AppImage and `.deb` |
