# Kyrei v0.4.7 — Open folder, stores ready

Patch release: after install, **open a project folder → local databases and memory index are ready**.  
No manual “Rebuild index”. Builds on v0.4.6 first-chat OOB.

---

## Highlights

| | |
| --- | --- |
| **Open workspace = ready** | Choosing a project folder **awaits** creation of `.kyrei` layout, SQLite memory index, project graph DB, and a warm reindex. |
| **Session mirror on start** | Gateway start creates the chat dual-write SQLite store under app data. |
| **Memory index reliability** | No full reindex on every chat turn; `project_index` flushes graph-lite into FTS before returning; file-backend fallback is durable on disk. |
| **macOS CI** | Pipeline temp-dir cleanup no longer flakes on `ENOTEMPTY`. |

---

## What’s fixed

| Area | Change |
| --- | --- |
| **Bootstrap** | `bootstrapGatewayLocalStores` + `bootstrapWorkspaceLocalStores` on start and workspace set |
| **Await open-folder** | `setConfig({ workspace })` and `/api/choose-folder` wait for DBs + reindex |
| **Infinite reindex** | `MemoryIndexSession` skips full projection when not dirty |
| **project_index → memory** | `flushMemoryIndex` awaited so search sees graph immediately |
| **entry candidates** | Incremental index includes `package.json` etc.; string paths in graph-lite |
| **File fallback** | Persists `memory-docs.json` / vectors so reopen still has docs |
| **Session reindex cap** | Settings rebuild projects at most 20 sessions (avoids multi-hour hangs) |
| **macOS gate** | Retry/tolerate temp `rm` races in pipeline tests |

---

## First launch (expected)

1. Install **v0.4.7** for your OS.
2. Open Kyrei → **add provider API key** (or local Ollama).
3. **Open a project folder**.
4. Confirm under the project: `.kyrei/memory/`, `.kyrei/index/`, `.kyrei/intel/project-graph.db`.
5. Chat and use memory tools — **no Settings → Rebuild required**.

### What still needs the user (not bugs)

| Item | Why |
| --- | --- |
| API key / local model | Chat needs a ready endpoint |
| Linux keyring | Secrets fail closed without Secret Service |
| Project folder | Workspace root for coding + project memory |

## Install

Verify artifacts against **`SHA256SUMS.txt`** in this release.

| Platform | Artifacts |
| --- | --- |
| Windows x64 | `Kyrei-Setup-0.4.7.exe`, `Kyrei-Portable-0.4.7.exe` |
| macOS | `Kyrei-0.4.7-x64` / `arm64` dmg+zip |
| Linux | AppImage, deb, **`Kyrei-0.4.7-*.pkg.tar.zst`** (Arch — use 0.4.6+ packaging rules) |

Prefer deb/pacman over AppImage. Never `sudo kyrei`. See [`docs/linux-launch.md`](../linux-launch.md).

## Upgrade notes

- From **0.4.6**: safe in-place. Open workspace once to ensure `.kyrei` stores exist.
