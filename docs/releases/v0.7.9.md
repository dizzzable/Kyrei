# Kyrei v0.7.9 — reliable agent terminals and first-pass guidance

This patch publishes the terminal lifecycle repair from the current mainline
with consistent package metadata, so installers and update checks identify the
same release source.

## Agent terminal reuse

- A later `run_command` from the same chat and agent actor reuses a finished
  terminal tab instead of accumulating dead tabs.
- Concurrent commands retain separate running tabs. Once a wave finishes,
  Kyrei keeps one free tab per actor and releases older finished tabs.
- Capacity recovery never reclaims a running process or a manual terminal.
  Failed process starts restore a reused or reclaimed tab together with its
  prior output.
- Agent commands no longer have a wall-clock kill. They end on process exit,
  user cancellation, or explicit terminal close.

## Better first responses

The provider-neutral system prompt preserves the prompt-improver principles
that fit Kyrei's architecture: clear requests proceed without ritual questions;
real ambiguity is grounded and resolved with a small number of concrete
choices; complex work is planned proportionately; delegated research returns
conclusions rather than raw output.

Kyrei does not embed Claude Code's hook implementation. Its static prompt and
native tool contracts remain provider-agnostic and do not depend on
Claude-specific hook events.

## Release integrity

- `package.json` and both root version fields in `package-lock.json` are now
  `0.7.9`.
- `npm run gate` verifies these fields remain aligned before typechecking and
  tests.
- The lockfile updates `fast-uri`, `postcss`, and `tar` to versions without
  the advisories reported by the previous dependency tree.
- The local `tmp/` smoke-workspace output is ignored and cannot accidentally
  enter a release commit.

## Verification

Before publishing, the release gate runs version consistency, engine and
renderer typechecks, JavaScript and i18n checks, and the full Vitest suite.
`npm audit --audit-level=high` reports no high-severity vulnerabilities.
The tag workflow additionally scans committed source for secrets, packages
Windows, macOS, Linux and Arch artifacts, and attaches SHA-256 checksums.

## Update

Install through Kyrei's updater when it offers v0.7.9, or download the
platform artifact from the GitHub Release. Existing sessions, provider
configuration, and local workspace data remain in place.
