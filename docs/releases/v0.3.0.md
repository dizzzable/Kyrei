# Kyrei v0.3.0 — команды агентов, провайдеры и настоящий desktop release

Это крупнейшее обновление Kyrei: приложение перешло от одиночного локального
агента к расширяемой рабочей среде с командами, pipeline-миссиями, памятью,
skills и управляемыми пулами провайдеров.

## Главное

- **Team mode и mission pipelines.** Независимые исследователи, исполнители,
  ревьюеры и тестировщики могут работать через общий оркестратор и проверяемые
  артефакты.
- **Пулы аккаунтов.** Балансировка, cooldown, session affinity, health state и
  назначение отдельных аккаунтов на конкретные модели.
- **Свободные провайдеры.** Готовый каталог и любое число собственных
  OpenAI-/Anthropic-/Google-совместимых подключений с auto-discovery моделей.
- **Официальный Kiro CLI connector.** Browser/device вход, Builder ID, GitHub,
  Google и IAM Identity Center, безопасный status boundary и обнаружение моделей.
- **Память и skills.** Проектные evidence/lease-механизмы, gbrain-интеграция,
  локальные skills и подготовка к дальнейшему auto-improvement.
- **Desktop UX.** Новый Hermes-подобный layout в визуальном языке Kyrei,
  исправленная геометрия окна и полноценная EN/RU локализация.

## Платформы

- Windows x64: Setup и Portable.
- macOS: отдельные DMG/ZIP для Intel и Apple Silicon.
- Linux x64/arm64: AppImage и DEB.
- Arch Linux x64: пакет `pkg.tar.zst` для `pacman`.

В релиз также входит `SHA256SUMS.txt` для проверки загрузок.

> Windows и macOS артефакты пока не подписаны коммерческими сертификатами.
> Сверяйте SHA-256 и ожидайте системное предупреждение при первом запуске.

## Безопасность и качество

- Credentials остаются в core и не передаются renderer'у.
- Вход выполняется системным браузером или device flow — встроенной страницы
  авторизации нет.
- Child-process окружение Kiro CLI ограничено allowlist'ом; вывод проходит
  bounded redaction.
- Полный release gate: TypeScript, JS/i18n проверки, Vitest, dependency audit и
  secrets scan перед нативными сборками.
