# Kyrei v0.7.8 — terminal reuse and no wall-clock shell kills

This patch stops agent shell commands from being killed by an arbitrary timer,
and makes agent/sub-agent terminal tabs reuse free slots instead of stacking
dead processes.

## Terminals

- Sequential `run_command` calls for the same actor reuse a **free** finished
  agent tab (same chat owner + actor id). Concurrent commands still get their
  own running tabs so processes stay isolated.
- After a wave of concurrent commands finishes, Kyrei keeps **one free tab per
  actor** for the next command and reclaims older finished same-actor tabs so
  capacity stays available.
- When the owner/renderer limit is reached, only **finished agent** tabs are
  reclaimed — never a running process and never a manual user terminal.
- Spawn failure while reusing a free tab no longer drops the prior tab or its
  output (claim happens only after a successful process start).
- Starting a new agent command focuses that tab in the UI (including in-place
  reuse of the same session id).

## No wall-clock kill

- Agent shell commands (desktop `commandRunner` and the in-process fallback)
  **wait until the process exits**.
- They are stopped only by **user cancel** (Stop / abort turn), **closing the
  terminal tab**, or a real process exit/error — not by a countdown timer.
- Long installs, tests, and builds are no longer cut off after 60 seconds.
- The old Settings “command timeout” control is removed; the config field remains
  for compatibility but is not used to kill live shells.

## Agent quality (prompt 1.35.x)

- First-pass guidance: clear vs vague asks, effort match, lean subagents,
  scannable deliverables.
- `run_command` policy text matches the new wait-until-exit behavior.
