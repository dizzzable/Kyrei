# Kyrei v0.4.4 — Linux secrets across desktops

Patch release: provider API keys on Linux/Arch work reliably when a Secret
Service keyring is available in the graphical session — including popular
Wayland and X11 environments.

---

## What’s fixed

| Area | Change |
| --- | --- |
| **Linux packaging** | Arch package advertises `optdepends` for `gnome-keyring`, `kwallet`, and `keepassxc`. DEB hard-depends on `libsecret-1-0` and recommends `gnome-keyring` + `libsecret-tools`. |
| **Session diagnostics** | When OS secret storage is unavailable, the desktop logs session type, desktop family, Electron backend, and install hints (no key material). |
| **UI guidance (EN/RU)** | Provider setup lists Arch, Debian/Ubuntu, and KDE install commands plus a pure-Wayland WM hint (Hyprland / Sway / niri / river). |
| **Docs** | [`docs/linux-launch.md`](../linux-launch.md) DE/WM matrix: GNOME, Plasma, Cinnamon, XFCE, MATE, Budgie, COSMIC, Pantheon, LXQt, Hyprland, Sway, niri, river. |

**Security unchanged:** Electron `basic_text` is still rejected. Keys are never
written as plain text when the keyring is missing.

---

## Desktop / keyring matrix

| Environment | Backend |
| --- | --- |
| GNOME, Cinnamon, XFCE, MATE, Budgie, COSMIC, Pantheon, LXQt | `gnome-keyring` |
| KDE Plasma | KWallet (`kwallet` / `kwalletmanager`) |
| Hyprland, Sway, niri, river | Install a Secret Service provider yourself, re-login, then `kyrei` |

Wayland is supported. “Secure storage unavailable” almost always means no
unlocked Secret Service in that session — not a Wayland ban.

```bash
# Arch (GNOME-family / most WMs)
sudo pacman -S gnome-keyring

# Arch (KDE)
sudo pacman -S kwallet

# Debian/Ubuntu
sudo apt install gnome-keyring
```

Then fully sign out and back in. Run `kyrei` as your normal user (never `sudo kyrei`).

## After install

1. **Settings → Providers** — add an API key.  
2. If secure storage fails on Linux: install the keyring for your DE, re-login, retry.  
3. Open a workspace → start a session.

---

## Upgrade notes

- From **0.4.3**: safe in-place upgrade.  
- No schema migrations. Existing encrypted secrets continue to work when the
  same OS keyring backend is available.  
- If you previously could not save keys on a minimal Wayland setup, install a
  Secret Service backend once and re-enter the key.
