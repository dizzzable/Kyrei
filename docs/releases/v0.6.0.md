# Kyrei v0.6.0 — reliable local memory and portable agent context

Kyrei v0.6.0 turns the memory workbench into a self-contained, local-first
part of the desktop agent. It removes the fragile first-run dependency on a
Bun-installed external memory CLI and makes imported agent context, memory
search, team storage, and update delivery more predictable.

## Highlights

| | |
| --- | --- |
| **Kyrei Memory, built in** | Personal memory now uses Kyrei's bundled SQLite/vector store. It works offline after explicit initialization, needs no Bun or GitHub runtime download, and keeps data separate from Team memory. An external GBrain CLI remains an opt-in advanced provider. |
| **Whole-history indexing** | Rebuilds and manual resynchronization no longer silently stop after a small session cap. The index follows the configured backend, including the built-in Team Postgres-compatible store. |
| **Import context from other agents** | The import pipeline recognizes Codex CLI JSONL, Hermes JSON and generic agent exports, normalizes model messages safely, and refuses malformed content instead of passing it to the model. |
| **Memory workbench** | Imported documents, auto-recall, and a navigable graph view make persistent project context visible instead of opaque. |
| **MCP over Streamable HTTP** | In addition to stdio, Kyrei can connect to Streamable HTTP MCP servers with protocol negotiation, timeout handling, sessions, and redacted diagnostics. Unsupported definitions remain visible instead of disappearing. |
| **Safer updates and prompts** | Updates install only after explicit confirmation; the effective system prompt now always retains its safety/communication core, bounds volatile project context, and exposes skill discovery consistently. |
| **Polished desktop flow** | Composer controls, microphone affordance, assistant-message chrome, localized settings, and default Team prompts have been aligned for a calmer day-to-day workflow. |

## Upgrade notes

- Safe in-place upgrade from **v0.5.2**. Existing sessions, provider secrets,
  project indexes, and Team data remain on disk.
- In **Settings → Memory**, select **Kyrei Memory (built in)** and initialize
  it once to enable local personal memory. The old default GBrain setting is
  migrated safely; a custom external command stays an explicit external CLI
  configuration.
- The built-in personal memory is deliberately not synchronized to Team mode.
  Team sharing continues to use its configured SQLite/Postgres backend.
- Updating replaces the packaged application only after confirmation. It does
  not delete session, secret, or memory directories.
- External MCP servers still require their own endpoint credentials and access
  policy. Kyrei does not ship or expose those credentials.
