# Kyrei v0.4.3 — Quality clock, tokens & knowledge

**Local-first multi-provider coding agent.** This release turns the agent into a **clockwork quality loop**: think → plan on disk → implement → verify → recover (3-strike) → final audit — while burning fewer tokens and growing skills **only with your approval**.

---

## What’s new (big picture)

```text
User goal
  → Coding mode (plan hard-gate / build / polish / deepreep)
  → Stable system prompt (prompt-cache friendly)
  → Tools with smart compression + read-memo
  → Phase VERIFY markers · 3-strike self-heal
  → Final audit (no empty green)
  → Optional skill sleep proposals · security/research packs · vault notes
  → Capacity failover · usage ledger · experimental gates
```

**Ship confidence:** full `npm run gate` green (**1447** tests) after multi-agent integration review. Empty final audits cannot mark a run complete.

## Wave A — Quality loop (like a clock)

| Piece | How it works now |
| --- | --- |
| **Karpathy discipline** | Always-on harness: think before coding, simplicity, surgical diffs, goal-driven checks (provider-agnostic). |
| **Run kit** | Namespaced long-horizon plans under `.kyrei/run/<id>/` — `ROADMAP`, `STATE`, `PROTOCOL`, `phases/phase-N.md`, optional `phase-N.fix.md`. Legacy `.kyrei/plan/` still works. |
| **Tools** | `run_claim`, `run_read`, `run_write_*`, `run_write_fix`, `run_phase_verify`, `run_final_audit` when planning is enabled (default on). |
| **Markers** | Transcript-friendly: `KYREI_PHASE_VERIFY`, `KYREI_FAILURE_PROBE/ESCALATE/HANDOFF`, `KYREI_FINAL_AUDIT`, `KYREI_RUN_COMPLETE`. |
| **3-strike heal** | Maps to self-heal FSM; handoff surfaces `KYREI_FAILURE_HANDOFF` + distilled handoff file. |
| **Final audit** | Re-scores criteria + commands + deliverables. **Empty evidence = fail** (`no_evidence_provided`) — never a false green. |
| **Plan mode** | Still hard-blocks edit/write/shell; plan/run kit writers stay allowed. |

Prompt version: **1.25.0**.

## Wave B — Tokens & reuse

| Piece | How it works now |
| --- | --- |
| **Tool compress** | Shape-aware shrink (JSON / logs / stacks / diffs / code / markdown) before the model pays tokens. Soft history prune still stores full bodies in **CCR** under `.kyrei/ccr/` → `retrieve(hash)`. |
| **Read-memo** | Same-turn `read_file` of an unchanged path@hash returns a short stub; writes/edits invalidate. |
| **Prompt cache packing** | System prompt split: **stable** harness vs **volatile** project context. Anthropic gets `cacheControl: ephemeral` on the stable system block. |
| **Web densify** | `web_fetch` prefers main/article → compact markdown; lower default body cap; SSRF boundary unchanged. |
| **Model roles** | Settings copy: worker = cheap, plan/build = strong, polish = strongest, deepreep = strong research + cheap fan-out. |

---

## Wave C — Skills & knowledge

| Piece | How it works now |
| --- | --- |
| **Skill sleep** | Offline harvest from recent sessions → **proposal-only** skill diffs (`skills-curator/sleep-*.json`). **Never** auto-writes `SKILL.md`. Settings → Skills → “Skill sleep”. |
| **Skill packs (opt-in)** | Built-in roots: **security** (`security-checklist`, `secret-hygiene`) and **research** (`research-first`, `deep-web-research`). Toggle adds a read-only root — no silent copy into your catalog. |
| **Markdown vault** | Optional absolute folders of personal notes → `memory_search` + index projection. Vault is **DATA**, not system policy. |

## After install (60 seconds)

1. **Settings → Providers** — API key or connector.  
2. Optional: **Capacity** (spare keys), **Usage** budgets, **Skills** packs / vault.  
3. Open a workspace → pick mode (or Auto) → give a goal.  
4. For long work: agent can `run_claim` and keep ROADMAP/STATE on disk across context resets.  
5. Before “done”: expect VERIFY / final audit evidence — not empty green checks.
