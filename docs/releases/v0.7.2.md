# Kyrei v0.7.2 — Continuous work, compact context

Kyrei 0.7.2 turns long conversations into durable, inspectable work rather
than a single provider-sized prompt. It also completes the project MCP path,
polishes desktop behaviour and makes custom provider connections practical
without weakening the network boundary.

## Continuous sessions and context

- A session can now be continued in a clean context window from the session
  menu. The complete chat remains preserved; the new session receives a
  bounded, redacted checkpoint instead of a duplicated transcript.
- Continuation checkpoints include the active task, rolling context summary,
  recent tool failures and completed `write_file` / `edit_file` receipts.
  Shell and MCP effects are never presented as verified file changes without
  evidence, and the agent is instructed to inspect the workspace before making
  a completion claim.
- The previous hidden 500-message retention limit is removed for new sessions.
  Compaction remains a provider-independent model projection: original chat
  records are not rewritten, large tool output and middle turns remain
  recoverable through local CCR artifacts.
- Context handoffs now understand the current AI SDK `input` tool shape and do
  not confuse an unfinished tool call with completed work.

## Project MCP and workspace boundaries

- MCP now has both global and project scope. Project servers live in
  `.kyrei/mcp.json`, need an explicit workspace-trust action and are merged
  predictably with the global list.
- Project MCP commands run from the selected Kyrei workspace and no longer use
  a neighbouring `.kiro` directory as their configuration or policy source.

## Runtime, providers and desktop polish

- Slow agents and delegated roles continue on observable progress rather than
  dying on an arbitrary timer; cancellation remains immediate and explicit.
- Custom provider routing accepts legitimate public-IP/self-hosted origins
  while preserving SSRF, redirect and DNS-rebinding protections.
- The Electron window controls now follow the selected application theme.
  Composer focus, Escape cancellation, image routing and update signalling have
  targeted reliability fixes.

## Verification

- `npm run gate`: 237 test files passed; 1,700 tests passed and 4 skipped.
- Engine and renderer typechecks, JavaScript checks and RU/EN i18n checks
  passed.
- Production renderer and engine preparation completed with
  `npm run package:prepare`.

## Upgrade notes

Install normally through Kyrei's updater or download the package for your
platform. Existing sessions, memories, provider secrets and workspace data are
preserved. Chats that were already truncated by older releases cannot recover
the deleted rows, but new and surviving sessions are no longer trimmed at 500
messages.
