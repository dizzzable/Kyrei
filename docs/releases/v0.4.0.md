# Kyrei v0.4.0 — правдивый desktop-workbench для команд агентов

Этот релиз превращает интерфейс Kyrei в полноценную рабочую среду: модельные
лимиты больше не угадываются, агенты показывают реальную активность процессов,
а команды получают проверяемые prompt-, skill- и checkpoint-контракты.

## Главное

- **Точные возможности моделей.** Контекст, максимальный вывод, модальности и
  функции привязаны к точному сочетанию provider, base URL, protocol и model.
  Результат auto-discovery подтверждается gateway receipt; неизвестные значения
  остаются неизвестными и могут быть переопределены пользователем.
- **Единая модель исполнения.** Ограничения применяются к основной, резервным,
  вспомогательным и Team-моделям без скрытого отката Team → Single.
- **Настоящие терминалы.** Можно открыть несколько независимых shell-процессов.
  Команда агента отображается в том же процессе, который действительно выполняет
  работу, с ограничением вывода, редактированием секретов и остановкой дерева
  процессов при отмене или тайм-ауте.
- **Рабочие пространства.** Нативный выбор папки и drag-and-drop канонизируют и
  проверяют путь в Electron main process до выдачи файловых возможностей.
- **Prompt profiles и skills.** Пользовательские системные дополнения назначаются
  главному агенту и ролям Team. Поддержаны локальные `~/.kiro/skills` и связанные
  документы из `~/.kiro/docs` с ленивым безопасным чтением без сырой инъекции в
  системный prompt.
- **Автоматические checkpoints.** Перед `write_file` и `edit_file` создаются
  ограниченные снимки; возврат к сообщению транзакционно восстанавливает файлы и
  состояние сессии либо откатывает неудачную попытку восстановления.
- **Наблюдаемость без имитации.** Reasoning показывается только тогда, когда его
  действительно передал провайдер. В интерфейсе видны сессии, терминалы, tool
  activity и свёрнутые diff-блоки добавленных и удалённых строк.
- **Организационный Kiro broker.** Изолированные профили, project/model policies,
  generation fencing, отзыв leases и защищённое хранение credentials отделены от
  обычной пользовательской Kiro-сессии.
- **EN/RU.** Новые экраны, состояния ошибок и действия локализованы без строк,
  зашитых напрямую в компоненты.

## Платформы

- Windows x64: Setup и Portable.
- macOS: DMG и ZIP для Intel и Apple Silicon.
- Linux x64/arm64: AppImage и DEB.
- Arch Linux x64: пакет `.pkg.tar.zst` для `pacman`.

Релиз содержит `SHA256SUMS.txt` для проверки каждого артефакта.

> Windows и macOS пакеты пока не подписаны коммерческими сертификатами и не
> notarized. Операционная система может показать предупреждение при первом
> запуске; сверяйте SHA-256 из релиза.

## Качество и безопасность

- Полный gate проходит на Windows, macOS и Ubuntu.
- Release pipeline повторяет typecheck, JS/i18n checks, Vitest и dependency audit,
  затем запускает gitleaks до нативной упаковки.
- Renderer не получает API-ключи, OAuth-токены или возможность самостоятельно
  пометить терминал как агентский.
- Discovery metadata не принимается по одному совпадению origin: gateway сверяет
  полный endpoint, protocol, model и ограниченный по времени receipt.

## Известные ограничения

- Интерактивные терминалы пока pipe-backed: PTY/ConPTY для полноэкранных TUI будет
  отдельным этапом.
- Автоматические checkpoints покрывают файловые write/edit tools; произвольные
  изменения файлов внутри `run_command` пока не перехватываются.
- Подпись Windows и notarization macOS требуют отдельных сертификатов владельца.
