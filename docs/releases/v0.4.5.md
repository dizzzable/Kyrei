## Install out of the box

### Requirements

| Platform | Notes |
| --- | --- |
| **Windows** | Setup or Portable; OS protected storage (DPAPI / Electron safeStorage) for API keys |
| **macOS** | dmg/zip; Keychain via Electron safeStorage |
| **Linux** | AppImage, deb, or pacman; **Secret Service** for saving keys (`libsecret` + keyring recommended) |
| **Arch + Wayland** | Package **must be 0.4.5+** (no broken `libappindicator` dep). Install `gnome-keyring` (or KWallet). Launcher sets `ELECTRON_OZONE_PLATFORM_HINT=auto`. See [`docs/linux-launch.md`](../linux-launch.md). |

### First launch (expected)

1. App opens **without** prior config or workspace.
2. Choose a **workspace folder** for coding / project memory tools.
3. Add a **provider API key** (fails closed if OS key storage is unavailable — no plaintext secrets).
4. Memory defaults are **on**: LTM, SQLite hybrid index, session mirror, curator (`apply_safe`), plan-as-files, reliability guards.

### Packaging chain (release)

```text
npm run package:prepare   # icons + vite renderer + engine .dist
electron-builder          # includes core/engine/.dist, unpacks better-sqlite3 / sqlite-vec
```

- Engine bundle is **required** for chat/tools (`core/engine/.dist/index.mjs` built in prepare, not git).
- CI packages on **native OS runners** (do not cross-pack Linux from Windows).
- Linux wrapper rejects root + `--no-sandbox` (security; use deb/pacman if AppImage namespaces are restricted).

### What needs user action (not bugs)

| Item | Why |
| --- | --- |
| Workspace folder | File tools, LTM, plan, memory index need a project root |
| API key + keyring | Chat needs a provider; secrets never stored plain |
| Vault / GBrain / MCP / OpenViking | Opt-in by design (off by default) |

## Config defaults

```yaml
memory:
  ltm: { enabled: true }
  index: { enabled: true, backend: sqlite, embed: { mode: lexical } }
  sessionMirror: { enabled: true, readSearch: true, enginePrimary: true }
  curator: { enabled: true, applyMode: apply_safe }
  recall: { mmrEnabled: true, clusterEnabled: true, k: 8 }
  decay: { enabled: true, floor: 0.05 }
  citeOrRefuse: { enabled: false }   # strict memory_search opt-in
```

Curator remains `apply_safe` by default (MEMORY.md not silently rewritten).
