# Kyrei v0.4.5 — Wave H memory (pin, SUPERSEDE, memory_ask)

Memory quality upgrade from MemoHood / MemoBase **patterns** (not hermes plugin dumps).  
Local-first, proposal-first for MEMORY.md, files remain source of truth.

---

## What’s new

| Area | Change |
| --- | --- |
| **LTM SUPERSEDE** | Atomic replace with history chain; `fetch_decision` + Settings history panel |
| **Pin** | Hard facts never decay in ranking; pin/unpin from Settings or `record_decision` |
| **Recall pipeline** | Near-dupe collapse + MMR on `memory_search` (offline) |
| **Capture signals** | Cheap RU/EN keyword scoring for curator (unicode-aware) |
| **memory_ask** | Grounded Q&A over vault + MEMORY/notes + decisions; refuse when weak |
| **cite-or-refuse** | Optional strict mode for `memory_search`; always on for `memory_ask` |
| **Settings** | Decisions browser; toggles for MMR / cluster / decay / strict search |

---

## Config defaults

```yaml
memory:
  recall: { mmrEnabled: true, clusterEnabled: true, k: 8 }
  decay: { enabled: true, floor: 0.05 }
  citeOrRefuse: { enabled: false }   # strict memory_search opt-in
```

Curator remains `apply_safe` by default (MEMORY.md not silently rewritten).
