# Security policy

## Reporting a vulnerability

Please use GitHub's private vulnerability reporting for
[`dizzzable/Kyrei`](https://github.com/dizzzable/Kyrei/security/advisories/new).
Do not open a public issue containing credentials, tokens, personal data or an
unpatched exploit.

Include the affected version, platform, reproduction steps, expected impact
and any suggested mitigation. Reports will be acknowledged as soon as
practical; release timing depends on severity and the availability of a safe
fix.

## Supported versions

Kyrei is currently pre-1.0. Security fixes are applied to the latest release.
Users should upgrade to the newest published version before reporting an issue
that may already have been corrected.
